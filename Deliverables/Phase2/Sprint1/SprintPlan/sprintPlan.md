# Sprint 1 Plan

- **Pipeline - DAST, SAST, IAST and SCA** integrate and run SAST (CodeQL/SpotBugs/PMD), IAST where available, and OpenAPI-driven DAST (OWASP ZAP). Owners: Miguel Cardoso (1220772), Pedro Soares (1200909) and Pedro Costa (1201576). 

- **Use Cases Implementation:** complete UC implementations (UC1, UC2, UC3 and UC4) per assignments above. Each UC owner delivers feature PRs with tests and ASVS evidence; testing lead consolidates test results. Owners: Diogo Ribeiro (1220812), Pedro Soares (1200909), Pedro Costa (1201576) and Pedro Silva (1221033)

- **Code Reviews:** enforce PR checklist (unit tests, integration tests, CI green, SAST/SCA triage, ASVS references). Review turnaround: 48 hours target; coordinators: every team member.

- **Database:** secure schema and migrations, use parameterized queries (JPA), enforce least-privilege DB user, and document backup/restore approach. Owner: Miguel Cardoso (122072).

- **Authentication and Authorization:** Auth0-based JWT validation, `AudienceValidator` enforcement, RBAC checks for admin endpoints, and token lifecycle settings (rotation/expiration). Owners: Pedro Soares (1200909).

- **Documentation:** Maintain and publish sprint documentation and evidence artifacts:
	 - ASVS traceability updates (`ASVSTraceability/Traceability.md`) and mapping notes.
	 - Pipeline and scanner runbooks (`PipelineAutomation/pipelineAutomation.md`) with masked-credential guidance.
	 - Test reports and coverage artifacts (`App/target/surefire-reports/`, `App/target/site/jacoco/`).
	 - Evidence assets (screenshots, exported reports) in `PipelineAutomation/assets/` and `images/`.
	 - Architecture diagrams and sequence diagrams (Phase 1 assets and Sprint 1 updates).
	 - Release notes and changelog references (generated by `release-please`).
	 Owners: Diogo Ribeiro (1220812), Pedro Costa (1201576) and Pedro Silva (1221033).
	 Acceptance: all links resolve, assets present, and at least one example scan/report reproduced locally.

 - **Unit and Integration Tests:** Ensure comprehensive automated test coverage for features and integrations.
	 - Scope: unit tests for service/utility classes, integration tests for controllers, persistence and security (JWT validation, RBAC). Include fault-injection tests for key failure modes.
	 - Frameworks/tools: JUnit 5, Spring Boot Test, MockMvc, Testcontainers or in-memory H2 for DB integration, Mockito for mocking, JaCoCo for coverage.
	 - Owners: Diogo Ribeiro (1220812), Pedro Costa (1201576) and Pedro Soares (1200909).

```bash
mvn -B -f App clean test jacoco:report
```

	 - Owners: `Miguel Cardoso` (tests lead), `Pedro Silva` (security test scenarios).
	 - Acceptance criteria:
		 - Unit tests present for all new/changed code paths and critical utilities.
		 - Integration tests exercise UC flows (login, purchase, refund) including auth and DB interactions.
		 - CI gate: tests must pass and JaCoCo coverage >= 70% for the `App` module (adjustable threshold).
		 - Test artifacts (surefire reports, jacoco HTML) published and linked in `TestingAndValidation/testingAndValidation.md`.

	 - Notes: use Testcontainers for realistic DB and external-service integration where possible; otherwise use in-memory DB with representative data fixtures.

## Collaboration Rules

- Open PRs for each feature; reference the responsible UC and evidence files.
- Track blockers in PRs or `issues.md`; tag owner and lead.
- PR checklist: tests present, CI green, SAST/SCA issues triaged, ASVS references added.
